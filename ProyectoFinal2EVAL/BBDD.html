¿Que se imparte en Bases de datos?

Modelo entidad-relación:

Un modelo entidad-relación es una herramienta para el modelo de datos, la cual facilita la representación de entidades de una base de datos.

Durante gran parte del primer trimestre, nos dedicamos a entender y realizar este tipo de modelos. Realizamos en clase ejemplos del tipo de la imagen y durante este tiempo se explica la importancia de una base de datos bien organizada.

Estos modelos podemos obtenerlos a través de ciertas reglas aplicadas a una descripción proporcionada en clase por el profesor, más tarde, en el mundo laboral estas descripciones serás proporcionadas por un cliente.

Formas normales:
Las formas normales protegen nuestras bases de datos de inconsistencias y anomalías lógicas, intentando perder la mínima información posible. En este curso aprenderemos las formas normales de la primera a la tercera y la FNBC ya que son las más comunes en la actualidad. Por supuesto conoceremos la importancia de la normalización y realizaremos practicas.
Puedes aprender más sobre formas normales pinchando en la imagen o dejar que nosotros le enseñemos. Para ello puede inscribirse para el curso que viene rellenando el siguiente formulario. Es muy sencillo, sólo introduzca sus datos y le llegará un correo electrónico para que se matricule cuando esté abierto el plazo de matrículas.
SQL-DDL:
Lenguaje de definición de datos (DDL): Proporciona órdenes para definir, modificar o eliminar los distintos objetos de la base de datos (tablas, vistas, índices...).
Durante gran parte del segundo trimestre vamos a dedicarnos a DDL, aprenderemos a crear tablas, editarlas, eliminarlas y rellenarlas con datos. Usaremos para ello oracle SQL con sql Developer para hacer más sencillo nuestro trabajo. Crearemos usuarios y conexiones para poder trabajar con cada uno de los enunciados que se propondrán en clase. Una de las partes más importantes de este apartado, será la capacidad de resolver errores que adquiriremos a base de práctica.

SQL-SELECT:

A lo largo de esta unidad nos centraremos en la cláusula SELECT. Sin duda es el comando más versátil del lenguaje SQL.
El comando SELECT permite:
-Obtener datos de ciertas columnas de una tabla (proyección).
-Obtener registros (filas) de una tabla de acuerdo con ciertos criterios (selección).
-Mezclar datos de tablas diferentes (asociación, join).

Los SELECT pueden ser complicados y enrevesados, pero son muy potentes.
Nos permiten acceder a los datos que tenemos almacenados en la base de datos y poder seleccionar exactamente los que queremos de entre todos los que tenemos.
